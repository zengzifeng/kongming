# 资源模型监控数据接口使用文档

## HTTP 接口

**请求方式**：`GET`

**请求路径**：`/api/v1/kingress/resource-model-monitor-data/list`

### 请求参数（Query 参数）

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| metrics | string[] | 否 | 返回全部 | 指定查询的监控项，为空返回全部。数组通过重复参数名传递，如 `?metrics=token_node_count&metrics=kingress_model_tpm` |
| model | string | 否 | `.+` | 模型名称筛选，同时作用于 Token 服务的 `inference_model` 和 Kingress 服务的 `ai_model` 标签（**推荐使用**） |
| inference_model | string | 否 | `.+` | 仅筛选 Token 服务的 `inference_model` 标签（优先级高于 model） |
| ai_model | string | 否 | `.+` | 仅筛选 Kingress 服务的 `ai_model` 标签（优先级高于 model） |
| ai_consumer | string | 否 | 空 | 消费者筛选，对应 Prometheus 中 `ai_consumer` 标签（仅 Kingress 服务指标有效） |
| start_time | string | 否 | 1小时前 | 开始时间，格式 `YYYY-MM-DD HH:MM:SS` |
| end_time | string | 否 | 当前时间 | 结束时间，格式 `YYYY-MM-DD HH:MM:SS` |

> **参数优先级**：`inference_model` > `model`（Token 服务），`ai_model` > `model`（Kingress 服务）。一般场景用 `model` 即可，无需区分。

### metrics 可选值

| 值 | 中文描述 | 输出精度 |
|---|---|---|
| token_node_count | token服务-模型节点数 | 整数 |
| token_cluster_tpm | token服务-模型集群TPM(万) | 整数 |
| token_node_avg_tpm | token服务-模型节点平均TPM(万) | 整数 |
| token_gpu_node_count | token服务-GPU型号对应台数 | 整数 |
| kingress_model_tpm | kingress服务-售卖模型TPM | 原始值 |
| kingress_thirdparty_ratio | kingress服务-第三方TPM分发占比 | 保留2位小数，百分制（如 75.00 表示 75%） |
| kingress_ksyun_ratio | kingress服务-自建TPM分发占比 | 保留2位小数，百分制 |
| kingress_avg_input_token | kingress服务-单次请求平均输入token | 整数 |
| kingress_avg_output_token | kingress服务-单次请求平均输出token | 原始值 |
| kingress_cache_hit_rate | kingress服务-缓存命中率 | 保留2位小数，百分制 |

### 请求示例

```bash
# 查询全部指标，最近1小时
curl "http://winlink.sre.ksyun.com/ksp_service/api/v1/kingress/resource-model-monitor-data/list"

# 指定模型和指标
curl "http://winlink.sre.ksyun.com/ksp_service/api/v1/kingress/resource-model-monitor-data/list?model=deepseek-v3.2&metrics=kingress_model_tpm&metrics=kingress_cache_hit_rate"

# 指定时间范围
curl "http://winlink.sre.ksyun.com/ksp_service/api/v1/kingress/resource-model-monitor-data/list?start_time=2024-01-15%2008:00:00&end_time=2024-01-15%2009:00:00"
```

## MCP Tool

**Tool 名称**：`kingress_resource_model_monitor_data`

**所属 Service**：`kingress`

**描述**：查询资源模型监控数据。包括 Token 服务（模型节点数、集群TPM、节点平均TPM、GPU型号台数）和 Kingress 服务（售卖模型TPM、第三方/自建TPM分发占比、单次请求平均输入输出token、缓存命中率）。默认返回最近1小时每分钟粒度数据。

### 参数说明

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| metrics | array | 否 | 指定查询的监控项，为空返回全部。可选: `token_node_count`, `token_cluster_tpm`, `token_node_avg_tpm`, `token_gpu_node_count`, `kingress_model_tpm`, `kingress_thirdparty_ratio`, `kingress_ksyun_ratio`, `kingress_avg_input_token`, `kingress_avg_output_token`, `kingress_cache_hit_rate` |
| model | string | 否 | 模型名称筛选，同时作用于 Token 服务的 `inference_model` 和 Kingress 服务的 `ai_model` 标签（**推荐使用**） |
| inference_model | string | 否 | 仅筛选 Token 服务的 `inference_model` 标签（优先级高于 model） |
| ai_model | string | 否 | 仅筛选 Kingress 服务的 `ai_model` 标签（优先级高于 model） |
| ai_consumer | string | 否 | 消费者筛选，对应 `ai_consumer` 标签（仅 Kingress 服务指标有效） |
| start_time | string | 否 | 开始时间，格式 `YYYY-MM-DD HH:MM:SS`，默认1小时前 |
| end_time | string | 否 | 结束时间，格式 `YYYY-MM-DD HH:MM:SS`，默认当前时间 |

### 调用示例

```json
// 查询 deepseek-v3.2 模型的售卖TPM和缓存命中率
{
  "model": "deepseek-v3.2",
  "metrics": ["kingress_model_tpm", "kingress_cache_hit_rate"]
}

// 查询全部指标，最近1小时
{}

// 分别指定 Token 和 Kingress 的模型筛选
{
  "inference_model": "qwen-72b",
  "ai_model": "qwen-72b",
  "start_time": "2024-01-15 08:00:00",
  "end_time": "2024-01-15 10:00:00"
}
```
